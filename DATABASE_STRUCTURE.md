# Database Structure & Specifications

This document describes the data persistence architecture of **SyllabiX**. It provides the database schema for the active SQLite instance (`curriculum_ai.db`) and a mapping guide for a Google Firebase Firestore and Cloud Storage implementation.

---

## Active SQLite Database Schema (`curriculum_ai.db`)

SyllabiX persists profile metadata and history logs locally using SQLite3. The database consists of two tables:

### 1. `users` Table
Stores account parameters, hashed passwords, credentials, and settings.
* **Fields & Data Types**:
  - `id`: `INTEGER` (PRIMARY KEY, AUTOINCREMENT)
  - `name`: `TEXT` (NOT NULL)
  - `email`: `TEXT` (NOT NULL, UNIQUE)
  - `password`: `TEXT` (NOT NULL)
  - `provider`: `TEXT` (NOT NULL, DEFAULT `'local'`) - Options: `'local'`, `'google'`
  - `photo_url`: `TEXT` (NULL) - Profile avatar URL
  - `firebase_uid`: `TEXT` (NULL) - Connected Firebase Auth identifier
  - `created_at`: `TEXT` (NOT NULL, DEFAULT `(datetime('now'))`)
  - `phone`: `TEXT` (NULL) - 10-digit number
  - `institution`: `TEXT` (NULL)
  - `role`: `TEXT` (NULL) - Academic designation
  - `bio`: `TEXT` (NULL)
  - `country`: `TEXT` (NULL)
  - `timezone`: `TEXT` (NULL)
  - `reset_token`: `TEXT` (NULL) - Token used for password reset flows

### 2. `history_records` Table
Stores curriculum blueprints and outcomes generated by users.
* **Fields & Data Types**:
  - `id`: `INTEGER` (PRIMARY KEY, AUTOINCREMENT)
  - `user_id`: `INTEGER` (NOT NULL, FOREIGN KEY referencing `users(id)` ON DELETE CASCADE)
  - `role`: `TEXT` (NOT NULL) - Options: `'faculty'`, `'student'`
  - `program_name`: `TEXT` (NOT NULL)
  - `domain`: `TEXT` (NOT NULL)
  - `industry_focus`: `TEXT` (NOT NULL)
  - `education_level`: `TEXT` (NOT NULL)
  - `semester_count`: `INTEGER` (NOT NULL)
  - `date_generated`: `TEXT` (NOT NULL)
  - `pdf_export_status`: `INTEGER` (NOT NULL, DEFAULT `0`) - Binary flag (`0` or `1`)
  - `json_export_status`: `INTEGER` (NOT NULL, DEFAULT `0`) - Binary flag (`0` or `1`)
  - `curriculum_data`: `TEXT` (NOT NULL) - Serialized JSON string containing the full curriculum object

## Firebase Firestore Conceptual Mapping

For a scalable cloud-first deployment, the SQLite schema and local storage objects can be migrated to Google Cloud Firestore collections:

### 1. `users` Collection
* **Document ID**: Auth user `uid`.
* **Fields**:
  - `name`: `string`
  - `email`: `string`
  - `phone`: `string`
  - `institution`: `string`
  - `role`: `string`
  - `bio`: `string`
  - `photoURL`: `string`
  - `createdAt`: `timestamp`
* **Relationships**: One-to-many relationship with `curriculums`, `quizzes`, and `history` collections.
* **Security Rules**:
  - Read/Write allowed only if authenticated and request `uid` matches document ID:
    ```javascript
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    ```

### 2. `curriculums` Collection
* **Document ID**: Auto-generated string UUID.
* **Fields**:
  - `userId`: `string` (Reference to `users` document)
  - `programName`: `string`
  - `educationLevel`: `string`
  - `industryFocus`: `string`
  - `weeklyHours`: `number`
  - `semesters`: `array` of semester objects
  - `stats`: `map` (totalCredits, totalUnits, totalTopics)
  - `createdAt`: `timestamp`
* **Security Rules**: Read and write allowed only if the authenticated user matches the `userId` field:
  ```javascript
  match /curriculums/{curriculumId} {
    allow read, write: if request.auth != null && resource.data.userId == request.auth.uid;
    allow create: if request.auth != null && request.resource.data.userId == request.auth.uid;
  }
  ```

### 3. `quizzes` Collection
* **Document ID**: Auto-generated string.
* **Fields**:
  - `curriculumId`: `string`
  - `userId`: `string`
  - `difficulty`: `string`
  - `questions`: `array` of question maps
  - `score`: `number`
  - `feedback`: `string`
  - `createdAt`: `timestamp`
* **Security Rules**: Auth user matches `userId`.

### 4. `studyPlans` Collection
* **Fields**:
  - `userId`: `string`
  - `curriculumId`: `string`
  - `planDuration`: `number` (e.g. 30, 60, 90)
  - `schedule`: `array` of day maps (tasks, completed flags)
  - `progress`: `number` (percentage integer)

### 5. `resources` Collection
* **Fields**:
  - `curriculumId`: `string`
  - `textbooks`: `array`
  - `guides`: `array`
  - `repos`: `array`

### 6. `history` Collection
* **Fields**:
  - `userId`: `string`
  - `role`: `string`
  - `programName`: `string`
  - `domain`: `string`
  - `dateGenerated`: `timestamp`
  - `payload`: `map`

### 7. `profiles` Collection
* **Fields**:
  - `userId`: `string`
  - `gamification`: `map` (score, badges array, completedUnitCount)

## Firebase Storage Specifications

Firebase Cloud Storage manages unstructured binary assets. Assets are partitioned under logical paths:

### 1. Profile Avatars
* **Path**: `/users/{userId}/avatar.png`
* **Purpose**: Custom profile avatar uploads.
* **Read Operations**: Public read allowed.
* **Write Operations**: Allowed only if user is authenticated and matches `userId`. Max size limit: 5MB.

### 2. Exported Handbook PDFs
* **Path**: `/users/{userId}/exports/{curriculumId}.pdf`
* **Purpose**: Cached versions of compiled curriculum booklets for fast distribution.
* **Access Rules**: Write allowed only for owner. Read allowed if shared or by owner.

### 3. DocMentor Uploaded PDFs
* **Path**: `/users/{userId}/tutor/documents/{documentId}.pdf`
* **Purpose**: Source text documents, study notes, or textbook materials uploaded by student users.
* **Constraints**: Format must be `application/pdf`. Max size limit: 50MB.
* **Access Rules**: Read/Write strictly restricted to the uploading user.
