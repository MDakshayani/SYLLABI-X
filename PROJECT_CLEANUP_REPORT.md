# Project Cleanup Report

This report documents unused files, redundant components, duplicate logic blocks, and code audit outcomes for the **SyllabiX** codebase. It provides recommended cleaning guidelines to optimize codebase maintenance.

---

## 1. Required Core Files
These files must not be deleted or modified as they contain critical platform routing, database tables, rendering algorithms, or global state stores.

* **Backend core**:
  - `backend/app.py`: Coordinates JWT authentication, profile edits, SQLite migrations, and history record logs.
* **Frontend layout and Routing**:
  - `src/App.jsx`: Registers React Router paths.
  - `src/main.jsx`: Initializes React 19 render trees.
  - `src/index.css`: Global styling variable declarations.
  - `src/components/Layout.jsx`: Core page dashboard structure.
* **Shared UI & Navigation**:
  - `src/components/NavigationHeader.jsx`: Sticky sub-navigation controls.
  - `src/components/PDFExportModal.jsx`: Generates print-ready files and manages download flows.
* **Utility layers & AI engines**:
  - `src/lib/pdfGenerator.js`: The A4 Academic Handbook PDF compiler.
  - `src/lib/docMentorService.js`: Parses uploaded notes and indexes text blocks for local semantic QA.
  - `src/lib/quizGroqService.ts`: Generates structured quizzes and evaluates student answers.
  - `src/lib/utils.js`: Custom curriculum outlining and fallback questions builder.
* **Zustand stores**:
  - `src/store/index.js` & `authStore.js`: Global state management.

---

## 2. Review Before Delete (Stubs & Legacy References)

The following files are referenced in the codebase but contain stub code, redirects, or graphics layers that could be refactored or simplified.

### 1. `src/pages/PDFPreview.jsx`
* **Current Purpose**: A minor routing wrapper that checks the store's active role and redirects `/curriculum/pdf` requests to `/curriculum` or `/dashboard`.
* **Recommendation**: If `/curriculum/pdf` route is no longer required on the frontend, remove this file and remove its route definition inside `src/App.jsx`.

### 2. `src/components/PremiumBackground.jsx` & `AboutBackground.jsx`
* **Current Purpose**: Particle effects and animated backgrounds.
* **Recommendation**: These components use HTML canvas rendering loops. If you want to optimize frontend CPU utilization, they can be replaced with simple static CSS gradients.

### 3. `src/components/NeuralCursor.jsx`
* **Current Purpose**: Tracks pointer interactions and animates a neural net vector trail.
* **Recommendation**: Safe to remove if user experience priorities change to native OS cursors.

---

## 3. Safe to Delete (Unused Assets & Code Blocks)

The following items are not referenced or have been superseded by cleaner implementations:

### 1. Unused Dev Dependencies in `package.json`
- **Tailwind Packages**: `tailwindcss` and `@tailwindcss/vite` are installed, but styling parameters are handled directly via custom Tailwind variables and vanilla classes defined inside `index.css`.
- **Action**: Unused Tailwind configuration lines can be removed if not planning to write utility class code.

### 2. Redundant Imports (Cleaned during recent audit)
- The dynamic CDN loading script for `pdf.js` inside `src/components/PDFExportModal.jsx` is no longer needed since in-app canvas previews have been removed.
- **Action**: Confirmed that the imports (`loadPdfJS`, `ZoomIn`, `ZoomOut`, `ChevronLeft`, `ChevronRight`, `ArrowLeft`, `Eye`) have been removed from `PDFExportModal.jsx`.

### 3. SQLite Database Backups (`backend/curriculum_ai.db-journal`)
- Temporary cache files generated by SQLite during transactions.
- **Action**: Safe to delete if the server is stopped; they will be recreated automatically during subsequent database writes.
