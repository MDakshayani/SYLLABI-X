# PDF Generation Engine Documentation

This document explains the PDF compilation engine of the **SyllabiX** platform. It describes how curriculum blueprints, study planners, and practice quizzes are rendered as professional, publication-grade academic handbooks.

---

## Technical Overview

SyllabiX renders PDF documents entirely on the client side using the **jsPDF** library (`jspdf`). 
The core implementation is managed within [pdfGenerator.js](file:///c:/Users/daksh/curriculum-ai/src/lib/pdfGenerator.js).

```text
[Zustand Store Data] ──► [pdfGenerator Exporters]
                                  │
                                  ▼
                     [Create jsPDF Document Object]
                                  │
                                  ▼
                     [Draw Professional Cover Page]
                       - Stacked Metadata Box
                       - Executive Summary
                                  │
                                  ▼
                   [Loop Semesters & Unit Outlines]
                     - Custom 'DocumentCursor'
                     - Page-boundary Calculations
                     - Blue Clickable Hyperlinks
                                  │
                                  ▼
                    [Add Header/Footer watermarks]
                                  │
                                  ▼
                   [Return Compiled doc & filename] ──► [Export Modal]
```

## Page Margins & Canvas Setup

The engine uses standard A4 size layouts in portrait mode with the following dimensions:
- **Unit Scale**: millimeters (mm)
- **Margins**: 0.75 inches (`19mm`) on all sides.
- **Content Width**: `172mm` (A4 width `210mm` - `38mm` total horizontal margin).
- **Content Height Bounds**: `35mm` to `265mm` (A4 height `297mm` - `38mm` total vertical margin).
- **Page Borders**: A thin decorative border is drawn on every page:
  `doc.rect(19, 19, 172, 259)`

---

## The `DocumentCursor` Helper Class

Since jsPDF does not feature an automatic HTML-style layout reflow, a custom tracking helper class called `DocumentCursor` coordinates vertical positioning:

### State Attributes
- `doc`: The underlying jsPDF instance.
- `title`: Running title string of the program.
- `margin`: Left margin offset (`19mm`).
- `width`: Total content width (`172mm`).
- `y`: Current vertical coordinate (advances dynamically).
- `pageNum`: Active page index.

### Key Methods
- `addPage()`: Adds a new page, increments `pageNum`, resets `y` to `35`, and draws headers/footers.
- `checkSpace(heightNeeded)`: Evaluates if `y + heightNeeded` exceeds the maximum threshold (`265mm`). If it does, automatically triggers `addPage()` to prevent text truncation.
- `writeTitle(text)`: Prints section titles in bold brand color and advances `y`.
- `writeSubHeader(text)`: Prints subheaders.
- `writeParagraph(text)`: Splices paragraph text using `doc.splitTextToSize` and draws it using `align: 'justify'` alignment.
- `writeBullet(text)`: Renders list items with hanging indents and bullet indicators (`·`).

## Header, Footer & Page Numbering

Running templates are drawn on every page (except the cover sheet) using the `drawHeaderFooter` method:
- **Separators**: Thin slate-grey horizontal rules at `y = 26` (Header) and `y = 270` (Footer).
- **Header Text**: Left: `SYLLABIX` | Right: `AI CURRICULUM GENERATOR` (font size: `8pt`).
- **Footer Text**: Left: `GENERATED BY SYLLABIX · CURRENT_DATE` | Right: `PAGE X` (font size: `8pt`).
- **Watermark**: A semi-transparent text string `GENERATED BY SYLLABIX` is drawn diagonally at a `45` degree angle across the center of each page (`x = 100`, `y = 150`, color: light grey).

## Cover Page Layout & Overlapping Fix

The cover page features:
1. **Program Header**: Main title in bold primary color (`32pt` scaled).
2. **Metadata Specification Box**: A light grey rectangle (`#F8FAFC`) at coordinates `x = 25`, width `160`, height `58`.
3. **Vertically-Stacked Specifications**:
   - Items are rendered in a 2-column grid. Column 1 starts at `x = 30`, Column 2 at `x = 110`.
   - The label (e.g. `COURSE TITLE`, `PROGRAM DURATION`) is rendered in a slate-grey bold font (size `8.5pt`).
   - The value is rendered vertically stacked below the label:
     ```text
     LABEL TEXT
     Value Text (automatically wrapped to a max width of 70mm)
     ```
   - Row offsets are set to `13mm` (Curriculum) or `18mm` (Planner) to prevent any text overlaps.
4. **Executive Summary**: A justified introductory description detailing workload hours, domains, and career alignments.

## Clickable Resource Hyperlinks

Curriculums contain clickable hyperlinks mapped directly into unit details and resource sections:
- Resources (YouTube, Google, research paper links, GitHub code starters) are rendered in a bold primary blue color.
- Interactive link zones are registered using jsPDF’s text links parameters:
  ```javascript
  doc.textWithLink(resource.name, x, y, { url: resource.url })
  ```
- This enables users to click references inside their exported PDF handbook and trigger automatic browser redirections.

## Export Process Flow

1. The user initiates a download inside the UI (e.g. clicks "Export PDF").
2. The UI invokes the respective generator:
   - `downloadCurriculumPDF(curriculum)`
   - `downloadStudentBlueprintPDF(blueprint)`
   - `downloadStudentPlannerPDF(planner)`
3. The generator compiles the jsPDF document object, structures all elements, and returns `{ doc, filename }`.
4. The frontend captures the output and opens the `PDFExportModal` confirmation window.
5. Clicking **Download PDF Document** invokes `doc.save(filename)` to download the file directly to the user's browser.
